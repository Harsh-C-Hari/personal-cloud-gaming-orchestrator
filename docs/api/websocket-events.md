# WebSocket Events

## Overview

WebSockets provide real-time communication between backend services and the dashboard.

The goal is to reduce polling requirements while keeping monitoring information current.

---

# Currently Implemented Usage

The WebSocket system is used to broadcast real-time platform updates to connected dashboard clients.

Current usage includes:

* Session state updates
* Dashboard status updates
* Monitoring updates
* Recovery-related updates

---

# Session Related Events

Examples from the session system include:

* session_started
* session_running
* session_cleaning
* session_completed
* session_failed
* warning_sent
* timer_expired

---

# Recovery Related Updates

Used to keep recovery information synchronized across the dashboard.

Examples include updates generated by:

* Sunshine watchdog activity
* Tailscale recovery activity
* Recovery event creation
* Recovery statistics changes

---

# Future Expansion

As the platform evolves, additional event categories may be introduced for:

* User application updates
* Authentication events
* Audit logging
* Advanced monitoring

These are not currently implemented.
